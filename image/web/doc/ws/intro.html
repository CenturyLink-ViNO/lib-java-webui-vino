All of the Web services accept and return both XML or JSON formatted data.

A sample invocation of a web service using the 'curl' tool could look like this:

         curl -k -v
            --header "Accept:application/json"
            -H "Content-Type:application/json"
            -X PUT
            -d @<datafilename>
            "http://<USERNAME>:<PASSWORD>@<HOST_ADDRESS>/ui/THE_DESIRED_SERVICE"
  • '<HOST_ADDRESS>' is the IP address or DNS name of the web service target device
  • '<USERNAME>:<PASSWORD>' is the web service user credentials used to log in to the target device
  • '/ui/DESIRED_SERVICE' is the web service URL to access
  • The Accept header controls the response type (JSON or XML)
  • The Content-Type header declares the format of the content in the data file you are PUTting (JSON or XML)
  • The '-X PUT' says "generate a HTTP PUT request" (the default is HTTP GET request) PUT, POST, GET, and DELETE are supported as defined by the documentation for the service you are invoking
  • '-d @<datafilename>' specifies the name of the file (for example 'sample.json') that contains the JSON data to send
  • The '-k' and '-v' are optional, used to show progress while debugging responses
